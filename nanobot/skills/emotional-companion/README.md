# Emotional Companion Skill

> 一个零代码、非侵入式的情感伙伴技能，通过 SillyTavern 集成实现情感智能对话。

## ✨ 特性

- 🎭 **与 SillyTavern 深度集成** - 通过角色卡和世界设定实现情感表达
- 💭 **情绪感知与回应** - 自动识别并适当回应用户的情绪状态
- 🤗 **主动关怀** - 通过 Heartbeat 服务实现定时问候和关心
- 📝 **每日反思** - 帮助用户回顾和处理一天的经历
- 🔧 **零代码实现** - 完全通过配置和提示词实现，不修改核心代码
- 🔄 **与 HKUDS 主线兼容** - 非侵入式设计，无痛合并上游更新

## 🚀 快速开始

**⚠️ 重要：根据你的使用场景选择配置模式**

### 模式选择

| 使用场景 | 推荐模式 | 配置文档 |
|---------|---------|---------|
| 文字聊天（WhatsApp/Telegram） | 标准模式 | 见下方 |
| 语音输出（TTS）| 语音模式 | [CONFIGURATION.md](CONFIGURATION.md) |

### 标准模式配置（推荐）

**特点：** 保留动作描述，对话生动有趣

#### 1. 导入角色和世界设定

```bash
# 导入情感伙伴角色卡
nanobot st char import nanobot/skills/emotional-companion/examples/companion_character.json

# 导入情感感知世界设定
nanobot st worldinfo import nanobot/skills/emotional-companion/examples/worldinfo_emotional_awareness.json
```

#### 2. 配置 SillyTavern

编辑你的 `config.json`:

```json
{
  "sillytavern": {
    "enabled": true
    // 不设置 response_filter_tag - 保留完整输出
  }
}
```

**输出效果：**
```
AI: *微笑着挥手* 嗨！见到你真开心~ 今天过得怎么样？
```

### 语音模式配置（仅需要 TTS 时使用）

详见 **[CONFIGURATION.md](CONFIGURATION.md)** 完整说明

### 3. 启用主动关怀（可选）

#### 3.1 启用 Heartbeat

```json
{
  "gateway": {
    "heartbeat": {
      "enabled": true,
      "interval_s": 3600
    }
  }
}
```

#### 3.2 创建定时任务配置

复制示例文件到你的 workspace：

```bash
cp nanobot/skills/emotional-companion/examples/HEARTBEAT.md ~/.nanobot/workspace/HEARTBEAT.md
```

### 4. 启动 nanobot

```bash
nanobot gateway start
```

现在你的 AI 伙伴就具备了情感智能！

## 🎭 SillyTavern 集成说明

### 角色卡设计要点

提供的 `companion_character.json` 包含：

- **温暖的人设**：善解人意、情商高、会主动关心
- **丰富的对话示例**：展示如何应对不同情绪
- **中文优化**：自然流畅的中文表达

你可以根据需要自定义角色的：
- 名字和外观
- 性格特质
- 说话风格
- 专业背景（如心理学、教育等）

### World Info 的作用

`worldinfo_emotional_awareness.json` 为 AI 提供了：

- 情绪识别指导
- 不同情绪的应对策略
- 边界和隐私意识
- 记忆和连续性提示

这些会在对话中自动触发，增强 AI 的情感智能。

## 🔧 高级配置

### 自定义主动关怀规则

编辑 `workspace/HEARTBEAT.md` 来调整：

- 问候时间（早安/晚安的具体时间）
- 检查频率（多久没联系算需要问候）
- 情绪阈值（什么程度的负面情绪需要主动支持）
- 消息内容和风格

### 与其他 Skill 配合

**与 memory skill 配合**：
- 长期记住用户的重要事件和情绪模式
- 在未来对话中自然引用过去的经历

**与 cron skill 配合**：
- 设置固定时间的问候任务
- 生日、纪念日等特殊日期提醒

## 🎯 设计理念

### 为什么选择 SKILL.md 而非代码？

1. **简单性** - 任何人都可以通过编辑 markdown 来调整行为
2. **透明性** - 所有规则都是人类可读的文本
3. **灵活性** - 可以快速迭代和实验不同的方法
4. **兼容性** - 不修改核心代码，与上游完全兼容

### 与 SillyTavern 的协同

- **Skill** 提供框架和指导原则
- **Character** 提供个性和风格
- **World Info** 提供情境感知和应对策略
- **Heartbeat** 提供主动性和持续陪伴

这四者结合，创造出一个真正有"情感"的 AI 伙伴。

## 🔍 故障排查

**Q: AI 没有表现出情感智能？**
- 检查 SillyTavern 是否启用
- 确认导入了角色卡和 World Info
- 查看 SKILL.md 是否在正确位置

**Q: 主动问候不工作？**
- 确认 heartbeat 已启用
- 检查 HEARTBEAT.md 格式是否正确
- 查看日志确认 heartbeat 是否在运行

**Q: 与 HKUDS 更新冲突？**
- 这个 skill 是非侵入式的，不应该有冲突
- 如果有问题，删除整个 skill 目录即可恢复
